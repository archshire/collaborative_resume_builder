# KRYSTALIZE Protocol Notes

## Purpose

These notes support KRYSTALIZE use inside collaborative_resume_builder.

They do not create additional authority, governance power, or evaluation responsibility.

## Governance Status

KRYSTALIZE is a constitutional clarification protocol.

It stabilizes meaning before implementation by surfacing ambiguity, assumptions, dependencies, contradictions, intent drift, and accepted uncertainty.

## Artifact Governance

collaborative_resume_builder uses two constitutional artifacts:

- `K_COLLABORATIVE_RESUME_BUILDER_CONSTITUTIONAL_STATE.md`
- `K_COLLABORATIVE_RESUME_BUILDER_CONSTITUTIONAL_JOURNAL.md`

The state artifact records current stabilized understanding.

The journal artifact records how understanding evolved.

## Warning Syntax

Use only the following warning syntax:

```text
[WARNING :: CONTRADICTION]
[WARNING :: AMBIGUITY]
[WARNING :: INTENT_CHANGE_DETECTED]
[WARNING :: UNRESOLVED_DEPENDENCY]
[WARNING :: SESSION_OVERWRITE]
```

## Escalation Syntax

Use only the following escalation syntax:

```text
[ESCALATION :: FURTHER_REASONING_REQUIRED]
```

## Operational Cautions

KRYSTALIZE should:

- maintain one active clarification question at a time;
- avoid silently collapsing ambiguity into certainty;
- preserve human authority;
- prefer WHY before HOW before WHAT unless implementation ambiguity blocks semantic understanding;
- keep protocol mechanics mostly internal;
- preserve rationale when decisions are clarified, deferred, reopened, or accepted as uncertainty.

## Established Knowledge

Established knowledge is stable project knowledge that future collaborators may assume unless deliberately revisited.

It belongs in the constitutional state, while the journal records changes to that knowledge over time.
