# collaborative_resume_builder North Star

## Purpose

collaborative_resume_builder is the individual-use Project Listen application for the 42 collaborative resume project.

It is a tool to help 42 students in Circle 6 enjoy the project without multitasking.

It helps a student turn an interview conversation into a professional resume draft and an evidence-backed skill profile.

The project exists to demonstrate disciplined AI-assisted software building, not merely to generate a resume.

## Core Philosophy

### Human Conversation First

The interview remains the source of truth.

The application should support the human interview process, not replace it.

### Evidence Before Claims

Resume content and skill-profile claims should be traceable to interview evidence whenever possible.

The AI should not invent achievements, experience, strengths, or interests that were not expressed or reasonably supported by the transcript.

### Clarity Before Automation

The first working version may keep transcription manual if that clarifies the user journey faster.

Automation should be added after the capture, transcript, resume, and skill-profile flow is stable.

## Product Intent

collaborative_resume_builder should allow a student to:

- record an interview in manageable audio chunks;
- preserve the recordings for later transcription;
- provide or generate a transcript;
- produce a professional resume draft from the transcript;
- generate an evidence-backed skill profile showing strengths, interests, job fit, and gaps.

## Success Criteria

The project succeeds when a reviewer can see:

- a working individual-use application;
- clear frontend and backend responsibility;
- responsible AI use;
- evidence-backed output;
- documented reasoning and architectural decisions;
- a path from manual proof of concept to automated transcription.
